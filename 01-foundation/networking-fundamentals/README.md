**什么是网络**：
- 计算机网络的定义

#### 1.2 OSI七层模型

**OSI模型结构**：
```
应用层 (Application)     - 为应用程序提供服务
表示层 (Presentation)    - 数据格式转换、加密
会话层 (Session)        - 建立和维护会话
传输层 (Transport)      - 端到端传输
网络层 (Network)        - 路由选择、IP地址
数据链路层 (Data Link)  - 帧传输、MAC地址
物理层 (Physical)       - 比特流传输
```

**学习重点**：
- 每层的功能和作用
- 每层对应的协议
- 数据封装和解封装过程

**学习资源**：
- OSI模型详解视频
- 在线动画演示

**实践任务**：
- 画出OSI模型图
- 理解数据在各层的封装过程

### 第2周：TCP/IP协议栈

#### 2.1 TCP/IP四层模型

**TCP/IP模型**：
```
应用层 (Application)    - HTTP、FTP、DNS、SMTP
传输层 (Transport)      - TCP、UDP
网络层 (Internet)       - IP、ICMP、ARP
网络接口层 (Link)       - Ethernet、Wi-Fi
```

**与OSI模型对比**：
- TCP/IP是实际使用的协议栈
- OSI是理论参考模型

#### 2.2 IP协议

**IP地址**：
- IPv4地址格式（32位，如192.168.1.1）
- IPv6地址格式（128位）
- 公网IP vs 私网IP
- IP地址分类（A、B、C、D、E类）

**子网划分**：
- 子网掩码
- CIDR表示法
- 子网划分计算

**实践任务**：
- 练习子网划分计算
- 理解IP地址规划

#### 2.3 TCP协议

**TCP特点**：
- 面向连接
- 可靠传输
- 流量控制
- 拥塞控制

**TCP三次握手**：
```
客户端                 服务器
  |                     |
  |-------- SYN ------>|  第1次：请求建立连接
  |                     |
  |<------ SYN,ACK ----|  第2次：确认并同意
  |                     |
  |-------- ACK ------>|  第3次：确认
  |                     |
```

**TCP四次挥手**：
```
客户端                 服务器
  |                     |
  |-------- FIN ------>|  第1次：请求关闭
  |                     |
  |<------ ACK --------|  第2次：确认
  |                     |
  |<------ FIN --------|  第3次：请求关闭
  |                     |
  |-------- ACK ------>|  第4次：确认
  |                     |
```

**学习资源**：
- TCP详解视频
- 《TCP/IP详解卷1》

**实践任务**：
- 使用Wireshark抓取TCP三次握手
- 分析TCP数据包结构

#### 2.4 UDP协议

**UDP特点**：
- 无连接
- 不可靠传输
- 速度快
- 适用于实时应用

**TCP vs UDP**：
| 特性 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠 | 不可靠 |
| 速度 | 较慢 | 快 |
| 应用 | HTTP、FTP、邮件 | DNS、视频流、游戏 |

### 第3周：应用层协议

#### 3.1 HTTP/HTTPS协议

**HTTP请求**：
```
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
```

**HTTP响应**：
```
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234

<html>...</html>
```

**HTTP方法**：
- GET：获取资源
- POST：提交数据
- PUT：更新资源
- DELETE：删除资源

**HTTPS**：
- SSL/TLS加密
- 证书验证
- 安全通信

**实践任务**：
- 使用浏览器开发者工具查看HTTP请求
- 使用curl发送HTTP请求

#### 3.2 DNS协议

**DNS解析过程**：
```
客户端
  ↓ (查询www.example.com)
本地DNS服务器
  ↓ (迭代查询)
根DNS服务器
  ↓
顶级域名服务器(.com)
  ↓
权威DNS服务器
  ↓ (返回IP)
客户端
```

**DNS记录类型**：
- A记录：域名→IPv4
- AAAA记录：域名→IPv6
- CNAME：别名记录
- MX：邮件交换记录
- TXT：文本记录

**实践任务**：
- 使用nslookup、dig查询DNS
- 理解DNS劫持原理

#### 3.3 其他协议

**FTP（文件传输）**：
- 控制连接：21端口
- 数据连接：20端口或随机端口
- 主动模式 vs 被动模式

**SSH（远程登录）**：
- 22端口
- 加密通信
- 密钥认证

**SMTP/POP3/IMAP（邮件）**：
- SMTP：发送邮件（25端口）
- POP3：接收邮件（110端口）
- IMAP：接收邮件（143端口）

### 第4周：网络工具与实践

#### 4.1 Wireshark抓包分析

**安装Wireshark**：
- Windows：官网下载安装
- Linux：`sudo apt install wireshark`

**基本使用**：
```
1. 启动Wireshark
2. 选择网络接口
3. 开始捕获
4. 过滤显示
5. 分析数据包
```

**常用过滤器**：
```
ip.addr == 192.168.1.1    # 过滤IP
tcp.port == 80            # 过滤端口
http                      # 只看HTTP
tcp.flags.syn == 1        # TCP SYN包
```

**实践任务**：
- 抓取HTTP请求和响应
- 抓取TCP三次握手
- 分析DNS查询过程

#### 4.2 网络诊断工具

**ping**：测试连通性
```bash
ping www.baidu.com
ping -c 4 192.168.1.1  # 发送4个包
```

**traceroute/tracert**：路由追踪
```bash
traceroute www.baidu.com
```

**netstat**：网络状态
```bash
netstat -an    # 查看所有连接
netstat -tuln  # 查看监听端口
```

**nmap**：端口扫描
```bash
nmap 192.168.1.1           # 扫描常见端口
nmap -p- 192.168.1.1       # 扫描所有端口
nmap -sV 192.168.1.1       # 服务版本识别
```

**实践任务**：
- 使用ping测试网络连通性
- 使用netstat查看网络连接
- 使用nmap扫描开放端口

#### 4.3 网络安全基础

**常见网络攻击**：
- **ARP欺骗**：中间人攻击
- **DNS劫持**：域名解析劫持
- **DDoS攻击**：分布式拒绝服务
- **端口扫描**：信息收集
- **嗅探**：流量窃听

**网络防御措施**：
- 防火墙
- IDS/IPS
- VPN
- 加密通信
